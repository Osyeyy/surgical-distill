# Activation-guided online distillation across mismatched architectures

Standard knowledge distillation trains a student on outputs generated by a teacher as a static, offline process. The student sees what the teacher said, never how to said it. 

This project does it differently: 

Both models run simultaneously, activations are compared layer by layer at every forward pass, and the student is trained on exact point of divergence, not the output, not the average loss across the sequence, but the specific layer where the student's 
computation drifted from the teacher's

teacher: (Qwen/Qwen3-4B) (dense)

# Why this is different:

Most distillation approaches:

```
Teacher generates dataset -> train student on it -> done
```

This approach: 

```
same input -> both models forward pass simultaneously
           -> compare activations at each layer
           -> find layer of maximum divergence
           -> backpropagate from that layer directly
           -> repeat
```

The training signal is dynaminc and localized. Instead of saying "the output was wrong, adjust everything" it says "layer 7 is where you lost the teacher's computation - fix layer7."

This is mode information-dense supervision and more computationally efficient than output-level KL.

# Components

1. Projection layer

   Bridges the hidden dimension gap (1536 -> 2560) for activation comparison. A learned linear layer trained jointly with the distillation loop.
   The projection itself encodes structural correspondence between the two architectures, it is not a throwaway component.
   
2. Layer correspondence map

   24 Student layers cannot map 1 to 1 to 32 teacher layers. The mapping is determined by gradient norm scoring on both models across a calibration set, student layer i is aligned to the teacher layer that is functionally most similar, not just positionally closest.
   Initial mapping uses proportional alignment (i * 32/34), then refined by activation similarity after the diagnostic phase.
   
3. Divergence localization

   At each forward pass, cosine similarity is computed between projected student activations and teacher activations at each aligned layer pair. The layer with minimum similarity is the training target for that sample. Loss is computed and backpropagated from that layer specifically.

4. MoE Handling
   Student model layers activate only ~1B of 3B parameters per token (top-4 of 64 routed experts + 1 shared). Activation comparison uses post-routing activations, what actually ran, not what could have run. Expert routing patterns are logged separately to track whether
   specific experts specialize toward teacher layer functions over training.

5. Online training loop

  for each batch:
    run teacher forward pass (no grad)
    run student forward pass (with grad)
    project student activations → teacher space
    compute per-layer cosine similarity
    identify divergence layer (argmin similarity)
    compute loss at divergence layer
    backpropagate
    log: divergence layer distribution, similarity scores, loss




# Phases

## Phase 0 - Diagnostic

Before any training, run both models on 1000 prompts, capture all activations, project to shared space, measure baseline cosine similarity per layer pair. Output as a divergence map showing which layers are naturally close and which are far apart. this shapes the layer correspondence map and sets the baseline to measure improvement against.

## Phase 1 - Projection training

Train only the projection layer on activation matching with frozen student weights. Establishes a reliable bridge before the student starts moving.

## Phase 2 - Online distillation 

Full loop with student weights unfrozen (LoRA). Divergence-localized loss. Log everything.

## Phase 3 - Evaluation

Benchmark on ARC Challenge and GSM8K against:
- base student (no distillation)
- standard KL distillation baseline
- this method

The question being answered: Does localized activation-guided supervision recover more capability per parameter than output-level distillation?


# What this is not

This is not a production training pipeline. It is an experiment to test whether activation-localized distillation across mismatched architectures (MoE  student, dense teacher) produces measurably better capability transfer than standard approaches.
Results will be honest, including null results if the method doesn't outperform the baseline. 

### Related work: 

ROME (Meng et al., 2022) — factual knowledge localized in MLP layers; original motivation for this project

TinyBERT — layer-wise distillation with attention and hidden state matching

MiniLLM — reverse KL distillation for LLMs

DistiLLM — adaptive online distillation

AdaLoRA — adaptive rank allocation across layers

None of these combine: online generation + activation-localized loss + mismatched architecture projection + MoE student. That combination is what this project explores.


### Background: 


This project grew out of a series of experiments on MLP layer behavior during fine-tuning:

Perplexity-weighted-selective-finetuning — perplexity filtering partially recovers benchmark degradation but cannot protect math reasoning layers

mlp-surgery — gradient norm scoring identifies damaged layers; restoring them recovers 53% of performance drop without retraining

layer-vision — real-time MLP activation visualizer; makes internal model computation visible

The observation across all three: where information lives in a model matters more than how much data you train on. This project applies that observation to distillation.
