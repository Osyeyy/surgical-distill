# Activation-guided online distillation (v1: Qwen2.5 7B → 3B)

Standard knowledge distillation trains a student on outputs generated by a teacher as a static, offline process. The student sees what the teacher said, never how to said it. 

This project does it differently: 

Both models run simultaneously, activations are compared layer by layer at every forward pass, and the student is trained on exact point of divergence, not the output, not the average loss across the sequence, but the specific layer where the student's 
computation drifted from the teacher's

**Teacher:** Qwen/Qwen2.5-7B-Instruct (loaded in 4-bit nf4, frozen — fits on 4090)

**Student:** [Malum0x/mlp-surgery-restored-top30](https://huggingface.co/Malum0x/mlp-surgery-restored-top30) — a Qwen2.5-3B-Instruct that was previously SFT-damaged and then restored via [mlp-surgery](https://github.com/Malum0x/mlp-surgery). Starting from this checkpoint rather than vanilla Qwen2.5-3B-Instruct because the prior project already brought the small student's reasoning back to base, and the natural next question is: with the floor recovered, can activation-localized distillation push it past base by transferring something only the larger teacher knows?

# Why this is different:

Most distillation approaches:

```
Teacher generates dataset -> train student on it -> done
```

This approach: 

```
same input -> both models forward pass simultaneously
           -> compare activations at each layer
           -> find layer of maximum divergence
           -> backpropagate from that layer directly
           -> repeat
```

The training signal is dynaminc and localized. Instead of saying "the output was wrong, adjust everything" it says "layer 7 is where you lost the teacher's computation - fix layer7."

This is mode information-dense supervision and more computationally efficient than output-level KL.

# Architecture (same family, different shape)

This is a same-family pair (both Qwen2.5, same SwiGLU + RoPE + GQA pattern), but the shape mismatch is non-trivial.

|                    | Teacher (Qwen2.5-7B-Instruct) | Student (restored_top30, Qwen2.5-3B) |
|--------------------|-------------------------------|--------------------------------------|
| hidden_size        | 3584                          | 2048                                 |
| num_hidden_layers  | 28                            | 36                                   |
| vocab_size         | 151,936                       | 151,936                              |
| activation         | SwiGLU                        | SwiGLU                               |
| RoPE               | yes                           | yes                                  |

Vocab is identical, embeddings and output logits are directly comparable. Everything in between requires learned projection and a layer correspondence mapping. 

Note the inversion: the student has more layers than the teacher (36 vs 28). The mapping is many-to-few rather than the usual few-to-many.

# Components

1. Projection layer

   Bridges the hidden dimension gap (2048 -> 3584) for activation comparison. A learned linear layer trained jointly with the distillation loop.
   The projection itself encodes structural correspondence between the two architectures, it is not a throwaway component.
   
2. Layer correspondence map

   36 student layers do not map 1 to 1 to 28 teacher layers. The mapping is determined by gradient norm scoring on both models across a calibration set, student layer i is aligned to the teacher layer that is functionally most similar, not just positionally closest.
   Initial mapping uses proportional alignment (`i * 28/36`), then refined by activation similarity after the diagnostic phase.
   
3. Divergence localization

   At each forward pass, cosine similarity is computed between projected student activations and teacher activations at each aligned layer pair. The layer with minimum similarity is the training target for that sample. Loss is computed and backpropagated from that layer specifically.

4. Online training loop

  for each batch:
    run teacher forward pass (4-bit, no grad)
    run student forward pass (LoRA, with grad)
    project student activations → teacher space
    compute per-layer cosine similarity
    identify divergence layer (argmin similarity)
    compute loss at divergence layer
    backpropagate through student LoRA only
    log: divergence layer distribution, similarity scores, loss




# Phases

## Phase 0 - Diagnostic

Before any training, run both models on 1000 prompts, capture all activations, project to shared space, measure baseline cosine similarity per layer pair. Output as a divergence map showing which layers are naturally close and which are far apart. this shapes the layer correspondence map and sets the baseline to measure improvement against.

## Phase 1 - Projection training

Train only the projection layer on activation matching with frozen student weights. Establishes a reliable bridge before the student starts moving.

## Phase 2 - Online distillation 

Full loop with student weights unfrozen (LoRA). Divergence-localized loss. Log everything.

## Phase 3 - Evaluation

Benchmark on GSM8K and ARC Challenge against three arms:

- **Baseline 1:** restored_top30 (no further training) — the student as it stands.
- **Baseline 2:** restored_top30 + standard KL distillation — output-level KL on the same teacher, same data, same compute budget.
- **Method:** restored_top30 + this activation-guided method.

The question being answered: Does activation-localized supervision push the student past base on top of an already-strong starting checkpoint, more efficiently than output-level distillation?


# What this is not

This is not a production training pipeline. It is an experiment to test whether activation-localized distillation produces measurably better capability transfer than standard approaches.
Results will be honest, including null results if the method doesn't outperform the baseline. 

# Roadmap (v2, v3)

v1 fixes one variable at a time: same family, dense → dense, ~2.3x scale gap, student that already had targeted surgery applied.

If the method works here:

- **v2:** dense teacher → MoE student (OLMoE-1B-7B candidate, or a small custom MoE). Question: does the method still work with routing dynamics? Plus expert specialization (which teacher layers do specific experts learn to track?).
- **v3:** cross-family or cross-scale. 30B → 3B, or Qwen teacher into Mistral / Llama student. The original cross-architecture vision.

### Related work: 

ROME (Meng et al., 2022) — factual knowledge localized in MLP layers; original motivation for this project

TinyBERT — layer-wise distillation with attention and hidden state matching

MiniLLM — reverse KL distillation for LLMs

DistiLLM — adaptive online distillation

AdaLoRA — adaptive rank allocation across layers

None of these combine: online generation + activation-localized loss + same-family-but-different-shape pair + a student that already had targeted weight surgery applied. That combination is what this project explores.


### Background: 


This project grew out of a series of experiments on MLP layer behavior during fine-tuning:

[Perplexity-weighted-selective-finetuning](https://github.com/Malum0x/Perplexity-weighted-selective-finetuning) — perplexity filtering does not protect reasoning during SFT. Standard SFT on conversational data shifts performance unevenly; filtering hurts both metrics. Honest negative result, kept open.

[mlp-surgery](https://github.com/Malum0x/mlp-surgery) — gradient norm scoring identifies damaged MLP layers. Restoring the top-30 most-damaged layers from base weights with no retraining fully recovers ARC and crosses base on GSM8K (+1.14 in re-run, +1.44 in original). The student in this project is one of the verified outputs.

[layer-vision](https://github.com/Malum0x/layer-vision) — real-time MLP activation visualizer; makes internal model computation visible

The observation across all four: where information lives in a model matters more than how much data you train on. Surgical-distill applies that to the question of cross-model capability transfer.
